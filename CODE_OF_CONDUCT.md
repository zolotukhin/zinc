# Code of Conduct

ZINC is a technical project, but the standard for behavior is still simple: be respectful, be direct, and help keep the project useful for other people.

## Expected behavior

People participating in this project are expected to:

- discuss ideas and code in good faith
- focus criticism on the work, not the person
- be respectful of different experience levels
- share benchmarks, bug reports, and technical claims honestly
- accept correction when a claim is wrong or incomplete
- avoid unnecessary hostility, dogpiling, or performative behavior

## Unacceptable behavior

The following are not acceptable in project spaces, including GitHub issues, pull requests, and community chat:

- harassment, intimidation, or personal attacks
- hateful or discriminatory language
- repeated bad-faith arguing or deliberate disruption
- doxxing or sharing private information without permission
- spamming, trolling, or derailing technical threads
- misrepresenting benchmark data, logs, or reproduction steps

## Scope

This applies to:

- GitHub issues
- pull requests
- code review discussions
- project chat spaces such as the ZINC Discord

## Enforcement

Project maintainers may remove comments, close discussions, reject contributions, or block participation when behavior is clearly harmful to the project or its contributors.

## Reporting

If you need to report a conduct problem, contact the project maintainer privately through GitHub or the project Discord rather than escalating the situation in a public thread.

## Notes

This document is intentionally short. The goal is not process theater. The goal is to keep collaboration technically productive and safe enough that people want to keep contributing.
